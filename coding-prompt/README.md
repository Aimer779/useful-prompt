结构化规则
1. PRD（项目需求文档）
- 应用概述
- 用户流程
- 技术栈&API
- 核心功能
- 范围外和范围内的项目
2. 应用流程文档
分解：
- 描述应用的每个界面
- 用户如何从一个界面移动到另一个界面
- 简单语言，没有项目符号
- 非常具体
3. 技术栈文档
- 所有软件包和依赖项
- API文档的链接
- 首选的库或工具（Supabase、Stripe、NextAuth）
4. 前端指南
- 字体
- 调色板
- 间距&布局规则
- 首选UI库或框架
- 图标集
5. 后端结构文档
- 数据库模式
- 认证逻辑
- 存储规则
- 任何边缘情况
6. 实现计划
- 列出50多个如果你手写代码会采取的具体步骤
